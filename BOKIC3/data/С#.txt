Как расшифровывается ООП:Объектно Ориентированное Программирование:1
Создание объекта на основе другого:наследование:2
Класс, который можно наследовать, но из которого нельзя создать экземпляр объекта:абстрактный:2
Тип связи при котором один из видов объектов не может существовать без основного объекта:композиция:3
Принцип при котором элементы изолируются для контроля доступа к ним:инкапсуляция:3
Тип-указатель на функции, который может ссылаться на методы с совместимыми сигнатурами:Делегат:3
Механизм для объектов, позволяющим уведомлять другие объекты об изменениях (множественное число):События:3
Область памяти, используемая для динамически выделяемых объектов:Куча:2
Преобразование значимого типа в ссылочный тип:Боксирование:3
Преобразование ссылочного типа обратно в его исходный значимый тип:Разворачивание:3
